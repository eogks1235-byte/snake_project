# Snake 강화학습 프로젝트

Snake 게임을 통해 강화학습(Reinforcement Learning)을 배워보기 위한 프로젝트.

사람이 규칙을 가르치는 게 아니라, AI가 수십만 번의 시행착오를 통해
"어떻게 움직이면 오래 살고 먹이를 많이 먹는지" 스스로 터득한다.

---

## 왜 Snake인가?

- 규칙이 단순해서 환경 구현이 쉽고 AI 추천으로 난도가 낮아 쉽게 도전해 보았습니다.
- 처음엔 쉽고(먹이 찾기), 갈수록 어려워짐(몸이 길어질수록 피하기 어려움)
- **단기 목표**(먹이 먹기)와 **장기 목표**(죽지 않기)가 충돌 → 흥미로운 학습 문제
- 강화학습 교과서에 자주 등장하는 대표적인 예제

---

## 기술 스택

| 기술 | 역할 |
|------|------|
| **PPO** (Proximal Policy Optimization) | 강화학습 알고리즘 |
| **MLP** (다층 퍼셉트론 256→128→행동) | 관찰 벡터를 받아 행동을 결정하는 신경망 |
| **A*** (에이스타) | 몸통을 피해 먹이까지 최단 경로를 계산하는 GPS 알고리즘 |
| **gymnasium** | RL 환경 표준 인터페이스 |
| **stable-baselines3** | PPO 구현체 |
| **PyTorch** | 신경망 학습 |
| **pygame** | 게임 렌더링 |
| **matplotlib** | 실시간 학습 그래프 |

---

## AI가 보는 정보 (20차원 벡터)

```
[0~2]   즉각 위험    직진 / 우 / 좌  바로 앞 1칸이 벽이나 몸통인가?
[3~5]   예측 위험    직진 / 우 / 좌  2칸 앞이 위험한가? (코너 함정 예방)
[6~9]   이동 방향    현재 뱀이 향하는 방향 (위/아래/왼/오른 중 하나만 1.0)
[10~13] 먹이 방향    먹이가 어느 쪽에 있는가? (위/아래/왼/오른)
[14~17] A* 추천      몸통을 우회한 최단경로의 다음 방향
[18]    경로 존재    A*가 먹이까지 도달 가능한 경로를 찾았는가?
[19]    경로 길이    A* 경로 길이 (짧을수록 1에 가까움)
```

## 보상 구조

| 상황 | 보상 |
|------|------|
| 먹이 획득 | +10 |
| 죽음 (벽 / 몸통 충돌) | -50 |
| 먹이에 가까워짐 | +0.1 |
| 먹이에서 멀어짐 | -0.1 |
| 매 스텝 생존 | -0.01 (빠른 먹이 탐색 유도) |

## 학습 흐름

1. 8개의 Snake 환경을 동시에 실행 (병렬 학습 → 데이터 수집 속도 8배)
2. 512 스텝마다 수집한 경험으로 신경망 가중치 업데이트
3. 10,000 스텝마다 현재 모델 평가 → 최고 모델 자동 저장
4. **목표 점수 달성 시 목표를 5점씩 자동 상향** (5 → 10 → 15 → ...)
5. `Ctrl+C`로 중단하면 `models/ppo_snake_final.zip` 저장

## 개발 과정 — 무엇을 왜 바꿨는가

### 알고리즘
| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| **DQN** | **PPO** | DQN은 학습이 불안정하고 느림. PPO는 안정적이고 병렬 환경과 잘 맞음 |

### 신경망 (관찰 → 행동 결정)
| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| **NatureCNN** (이미지 처리) | **커스텀 SnakeCNN** | NatureCNN은 24×24 격자에 커널이 너무 커서 오류 발생 |
| **SnakeCNN** (24×24 이미지) | **MLP 128→128** (11차원 벡터) | CNN은 수백만 스텝이 필요. MLP + 벡터 관찰이 훨씬 빠르게 학습 |
| **MLP 128→128** | **MLP 256→128** | 입력이 11→20차원으로 늘어나 표현력 확대 |

### 관찰 공간 (AI에게 무엇을 보여주는가)
| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| 3채널 uint8 이미지 (3, 24, 24) | **11차원 벡터** (위험/방향/먹이) | 픽셀에서 방향·위험을 추론하는 건 CNN에게 너무 어려움 |
| 11차원 벡터 | **20차원 벡터** (+2칸 예측 위험 +A*) | 코너 함정 예방 및 몸통 우회 능력 추가 |

### 보상 구조
| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| 먹이 +1 / 죽음 -1 | 먹이 **+10** / 죽음 **-10** | 보상 스케일이 너무 작아 학습 신호가 약함 |
| 거리 보상 **±0.1** | 거리 보상 제거 | 매 스텝 -0.1이 누적되어 CSV 평균 점수 -9.35로 왜곡 |
| 거리 보상 없음 | 거리 보상 **±0.1** 재도입 | 보상 없이는 먹이를 찾기 전에 루프 방지 타임아웃으로 죽음 |
| 죽음 **-10** | 죽음 **-50** | 죽음과 먹이가 같은 가중치 → AI가 죽음을 두려워하지 않음 |
| 스텝 패널티 없음 | 스텝 패널티 **-0.01** | 2×2 원 돌기가 0 보상이라 AI가 제자리를 계속 돔 |

### 학습 환경
| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| 단일 환경 1개 | **병렬 환경 8개** | 데이터 수집 속도 8배, 더 다양한 경험 |
| 고정 목표 점수 | **목표 달성 시 +5 자동 상향** | 목표 달성 후 학습이 멈추는 문제 해결 |

### 로깅 / 모니터링
| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| TensorBoard | **CSV + matplotlib** | TensorFlow 로드 시 수십 초 지연 발생 |
| 누적 보상 `ep["r"]` 기록 | **실제 게임 점수** `info["score"]` 기록 | 거리 보상이 섞인 누적값은 실제 성능과 무관 |
